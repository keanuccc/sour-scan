📌 概述

Sour Scan 是一个基于 AI Agent 的全自动信息扫描与洞察系统。每天 21:00（CST）自动运行，从 19 个信源采集 AI 行业动态，生成结构化日报，并进行跨源分析与趋势追踪，最终通过飞书交付给用户。

你每天的 AI 行业情报官，自动扫描、自动分析、自动交付。

核心理念

  • 不遗漏关键信号 — 覆盖 19 个信源，从大厂动态到开源项目，从学术论文到社区讨论
  • 不被噪声淹没 — 自动去重、评分、分级，只推送真正值得关注的信息
  • 不只看今天 — 基于历史数据追踪话题热度变化，识别趋势而非孤立事件
  • 可落地 — 从洞察到建议，为 AI 产品经理的日常决策提供信息支撑

阶段划分

Sour Scan 的执行分为三个连续阶段：

阶段 1：信息搜集 & 日报生成

  1. 批量聚合 — 使用 fetch_news.py 从 6 个主流信源批量获取（Hacker News、GitHub、36氪、Product Hunt、HF Papers、华尔街见闻）
  2. AI 热点 — 抓取 aihot.virxact.com 获取当日热门
  3. 行业搜索 — 使用 web_search 在 TechCrunch、The Verge、Reuters、虎嗅等站点搜索 AI 相关文章
  4. 学术搜索 — 搜索 ArXiv、Papers With Code、Semantic Scholar
  5. 社区搜索 — 搜索 Reddit、知乎、微信公众号
  6. 汇总去重 — 将 19 个来源的内容去重后保留 10-15 条最有价值的信息
  7. 创建日报 — 调用飞书文档 API 创建并写入日报内容

阶段 2：分析与洞察

  1. 读取日报 + 历史数据（topic_trends.json、signals.json、insight_history.json）
  2. 四维分析：
    • 话题热度评分（模型发布 / 融投资 / 产品更新 / 开源项目 / 政策监管）
    • 趋势追踪（对比历史，识别升温 / 降温 / 新出现 / 持续话题）
    • 跨源关联发现（同一事件多源出现 → 信号确认）
    • 信号检测（watchlist 关键词覆盖统计）
  3. 生成洞察文档：包含概览、关联发现、趋势追踪、信号检测、关注建议
  4. 更新本地数据文件：topic_trends.json + insight_history.json
  5. 更新月度索引文档

阶段 3：交付

  • 今日扫描摘要（5-10 条最值得看的内容）
  • 洞察速览（关键信号、趋势、建议）
  • 链接到完整日报 + 洞察分析文档 + 索引

✨ 设计原则

质量优先

  • 宁缺毋滥，不够 5 条就不凑数
  • 每条信息标注来源，附带价值点分析
  • 来源不可达时优雅跳过，不报错不打断流程

容错优先

  • 三级降级策略确保数据获取的鲁棒性
  • web_fetch 失败 → Agent Browser → r.jina.ai → 跳过

长期积累

  • 话题热度数据持续积累，支持趋势分析
  • 每天的分析结果沉淀到 insight_history，形成知识库
  • 月度索引文档便于回顾

人类可读

  • 最终输出是结构化消息，不是原始数据 dump
  • 分析结果以人类可理解的方式呈现
  • 关注建议具有可操作性
